List of tags(HTML5):

  1. article - задає зміст сайту типу новини, статті, записи блогу, форуму або ін.

  2. aside - визначає блок збоку від контенту для розміщення рубрик, посилань на архів, міток та іншої інформації. Такий блок, як правило, називається «сайдбар» або «бічна панель»

  3. audio - додає, відтворює і управляє настройками аудіозаписи на веб-сторінці. Шлях до файлу задається через атрибут src або вкладений тег source. Усередині контейнера audio можна написати текст, який буде виводитися в браузерах, які не працюють з цим тегом.

  4. canvas - cтворює область, в якій за допомогою JavaScript можна малювати різні об'єкти, виводити зображення, трансформувати їх і міняти властивості. За допомогою тега canvas можна створювати малюнки, анімацію, ігри та ін.

  5. command - cтворює команду у вигляді перемикача, прапорця або звичайною кнопки. Тег command повинен розташовуватися усередині menu, в іншому випадку він не буде показаний

  6. datalist - cтворює список варіантів, які можна вибрати при наборі в текстовому полі. Спочатку цей список прихований і стає доступним при отриманні полем фокусу або наборі тексту.

  7. details - використовується для зберігання інформації, яку можна приховати або показати на вимогу користувача. За умовчанням вміст тега не відображається, для зміни статусу застосовується атрибут open.

  8. figcaption - містить опис для тега figure. Тег figcaption повинен бути першим або останнім елементом у групі.

  9. figure - використовується для групування будь-яких елементів(як контейнер), наприклад, зображень та підписів до них.

  10. footer - задає «підвал» сайту або розділу, в ньому може розташовується ім'я автора, дата документа, контактна і правова інформація.

  11. header- задає «шапку» сайту або розділу, в якій зазвичай розташовується заголовок.

  12. hgroup - використовується для групування заголовків веб-сторінки або розділу. Усередині розташовуються теги заголовків від h1 до h6. Даний тег виключений з версії HTML від W3C, але залишився у версії WHATWG, майбутнє елемента поки не визначено.

  13. keygen - використовується для генерації пари ключів - закритого і відкритого. Коли форма відправляється на сервер, закритий ключ зберігається на локальному комп'ютері, а відкритий ключ передається разом з формою. Самі ключі необхідні для шифрування і розшифровки даних, створення та перевірки цифрового підпису.

  14. main - призначений для основного вмісту документа. Вміст має бути унікальним і не включати типові блоки начебто шапки сайту, підвалу, навігації, бічній панелі, форми пошуку і т.

  15. mark - позначає текст як виділений. Такий текст нічим не відрізняється від звичайного, але його вигляд може бути змінений за допомогою стилів.

  16. menu - він виступає контейнером для елемента command і створення меню.

  17. meter - Використовується для виведення значення в деякому відомому діапазоні. Застосовується переважно для відображення числових значень, наприклад, кількості результатів пошуку, об'єму рідини, тиску та ін. Браузери, які підтримують елемент meter, відображають значення у вигляді малюнка, на якому кольором позначається поточне значення в зазначеному діапазоні. Решта браузерів виводять текст, написаний всередині meter.

  18. nav - eлемент nav задає навігацію по сайту. Якщо на сторінці кілька блоків посилань, то в nav зазвичай поміщають пріоритетні посилання.

  19. output - визначає область в яку виводиться інформація, переважно за допомогою скриптів.

  20. progress - використовується для відображення прогресу завершеності завдання. Зміна значення відбувається через JavaScript. Вид елемента залежить від браузера та операційної системи і може досить сильно відрізнятися між собою.

  21. rp - використовується для виведення тексту в браузерах, які не підтримують елемент ruby. В інших браузерах текст, укладений в контейнер rp, не відображається. Елемент rp повинен йти до або після rt.

  22. rt - eлемент rt додає анотацію зверху чи знизу від тексту, укладеного в контейнер ruby. Сама анотація виводиться зменшеним шрифтом.

  23. ruby - eлемент ruby призначений для додавання невеликої анотації зверху чи знизу від заданого тексту. Така форма запису переважно використовується для идеографической писемності начебто китайської мови, але може застосовуватися і для інших мов, якщо потрібно написати один текст над іншим. Сам елемент ruby виступає контейнером для rt, він і формує анотацію до тексту, після якого йде; а також rp, цей елемент призначений для браузерів, які не підтримують ruby.

  24. section - задає розділ документа, може застосовуватися для блоку новин, контактної інформації, глав тексту, вкладок в діалоговому вікні та ін. Розділ зазвичай містить заголовок.

  25. source - вставляє звукової або відеофайл для елементів audio і video. Узагальнено такі файли називаються медійними.

  26. summary - вказує заголовок для details, по якому можна клацати для розгортання/згортання інформації. Елемент summary повинен йти першим всередині details.

  27. time - позначає текст всередині елемента time як дата, час або обидва значення. Може вказуватися безпосередньо всередині контейнера time, або задаватися через атрибут datetime.

  28. video - додає, відтворює і управляє настройками відеоролика на веб-сторінці. Шлях до файлу задається через атрибут src або вкладений елемент source.

  29. wbr - вказує браузеру місце, де допускається робити перенесення рядка в тексті, якщо цього вимагає ширина батьківського елементу.
